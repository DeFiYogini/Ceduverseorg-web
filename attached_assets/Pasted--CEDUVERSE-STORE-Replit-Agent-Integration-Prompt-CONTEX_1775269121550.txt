# CEDUVERSE STORE — Replit Agent Integration Prompt

## CONTEXT
Ceduverse is a React 19 + Express + Drizzle ORM + PostgreSQL EdTech platform at ceduverse.replit.app. We need to add a **Hardware Store** (Vault Kit + Tangem wallets) with real payment processing (MercadoPago) and shipping (Envia.com).

I'm providing 4 files that contain ALL the code — your job is to integrate them into the existing codebase:
1. `store-schema.ts` — 7 Drizzle ORM tables (products, stock, orders, items, shipments, referrals, referral_uses)

3. `seed-store.sql` — Initial product catalog, stock quantities, and sample referral codes
4. The existing `ceduverse-medios-de-pago.jsx` landing page (reference for UI/UX, adapt to TSX)

- `store_products` — Product catalog with slug, price, weight, dimensions, seed phrase options
- `store_stock` — Real-time inventory with quantity, reserved, low_stock_threshold

Run `npx drizzle-kit generate` and `npx drizzle-kit migrate` after adding the schema.

## TASK 2: API Routes

Add the store API routes from `store-routes.ts` to the Express backend:

### In your main server file (e.g., `server/index.ts` or `server/routes.ts`):
```typescript
import storeRoutes from "./store-routes";
app.use("/api/store", storeRoutes);
```

### Install MercadoPago SDK:
```bash
npm install mercadopago
```

### Add environment variables to `.env`:
```
MP_ACCESS_TOKEN=APP_USR-xxxxx  (get from mercadopago.com.mx/developers/panel/app)
ENVIA_API_KEY=xxxxx
BASE_URL=https://ceduverse.replit.app
```

### The routes provide:
**Public:**
- `GET /api/store/products` — Product catalog with live stock
- `POST /api/store/validate-referral` — Check referral code validity
- `POST /api/store/shipping-quote` — Get Envia.com shipping rates
- `POST /api/store/create-order` — Create order + MercadoPago preference → returns `init_point` URL
- `POST /api/store/webhook` — MercadoPago payment notification (auto-confirms stock, generates shipment)

**Admin (protect with existing auth middleware for super_admin/admin roles):**
- `GET /api/store/admin/stats` — Dashboard stats (orders, revenue, referrals)
- `GET /api/store/admin/orders` — All orders (newest first)
- `GET /api/store/admin/orders/:id` — Order detail with items + shipment
- `PATCH /api/store/admin/orders/:id/status` — Update order status
- `GET /api/store/admin/stock` — Current stock levels
- `PATCH /api/store/admin/stock/:productId` — Update stock quantity / sold out
- `GET /api/store/admin/referrals` — All referral codes
- `POST /api/store/admin/referrals` — Create new referral code
- `DELETE /api/store/admin/referrals/:id` — Deactivate referral code

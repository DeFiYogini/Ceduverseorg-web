```
CONTEXT: I'm building a CRM/lead intelligence system for Ceduverse (EdTech STPS Mexico). 
Stack: React 19 + Express + Drizzle ORM + PostgreSQL + Vite + Tailwind.
The "Prospectos DENUE" section already exists in the admin dashboard.

TASK: Create/update the database schema for the full CRM system. Use Drizzle ORM.

Create these tables (if they don't already exist — check first):

1. `denue_prospects` — raw DENUE data + computed fields
   - id: serial primary key
   - clee: varchar (DENUE unique ID)
   - nom_estab: varchar (business name)
   - raz_social: varchar (legal name)
   - codigo_act: varchar (SCIAN code)
   - nombre_act: varchar (SCIAN activity name)
   - per_ocu: varchar (employee range string from DENUE)
   - employee_min: integer (parsed minimum, e.g. "11 a 30" → 11)
   - employee_max: integer (parsed maximum)
   - employee_midpoint: integer (midpoint for scoring)
   - full_address: text (concatenated from components)
   - tipo_vial: varchar
   - nom_vial: varchar
   - numero_ext: varchar
   - numero_int: varchar
   - nomb_asent: varchar (colonia)
   - cod_postal: varchar
   - municipio: varchar
   - localidad: varchar
   - entidad: varchar (defaults to "Nuevo León")
   - telefono: varchar
   - correoelec: varchar
   - www: varchar
   - latitud: decimal(10,6)
   - longitud: decimal(10,6)
   - fecha_alta: varchar
   - tipo_uni_eco: varchar
   - lead_score: integer (0-100, computed)
   - score_size: integer (component)
   - score_risk: integer (component)
   - score_digital: integer (component)
   - score_economic: integer (component)
   - priority: varchar ('Alta','Media','Baja')
   - risk_level: varchar ('Alto','Medio','Bajo')
   - sector_group: varchar (simplified sector from SCIAN)
   - commercial_zone: varchar (Centro, Valle Oriente, Norte, etc.)
   - estimated_contribution_monthly: decimal(10,2) (in MXN)
   - recommended_plan: varchar ('Impulsa','Transforma','Lidera')
   - applicable_noms: text (JSON array of applicable NOMs)
   - pipeline_stage: varchar (null=prospect, 'Nuevo','Contactado','Demo','Propuesta','Negociación','Cliente','Perdido')
   - assigned_socio_id: integer (references users table)
   - contact_group_id: integer (references contact_groups)
   - notes: text
   - last_contacted_at: timestamp
   - created_at: timestamp default now()
   - updated_at: timestamp default now()

2. `contact_groups` — named groups for campaigns
   - id: serial primary key
   - name: varchar (e.g. "Manufactura MTY 50+")
   - description: text
   - filter_criteria: jsonb (saved filter state)
   - assigned_socio_id: integer
   - prospect_count: integer (denormalized count)
   - created_at: timestamp default now()
   - updated_at: timestamp default now()

3. `prospect_interactions` — interaction history
   - id: serial primary key
   - prospect_id: integer references denue_prospects
   - interaction_type: varchar ('call','email','visit','demo','note')
   - description: text
   - performed_by: integer (user id)
   - created_at: timestamp default now()

4. `saved_filters` — reusable filter combinations
   - id: serial primary key
   - name: varchar
   - filter_config: jsonb
   - created_by: integer
   - created_at: timestamp default now()

After creating the schema, run the migration. Don't drop any existing tables.

Add indexes on: lead_score, priority, municipio, commercial_zone, sector_group, 
pipeline_stage, assigned_socio_id, contact_group_id, codigo_act.
```

## PHASE 1: CSV IMPORT + SCORING ENGINE
**Paste after Phase 0 completes. This loads and scores all DENUE data.**

```
CONTEXT: Ceduverse CRM. The denue_prospects table now exists (Drizzle ORM + PostgreSQL).
The DENUE CSV for Nuevo León is already in the project (find it — likely named 
denue_inegi_19_.csv or similar in the root or data folder).

TASK: Build the CSV import + scoring pipeline as a server-side script AND an API endpoint.

STEP 1 — Find and read the DENUE CSV file in the project. Identify the exact filename and path.

STEP 2 — Create a script `scripts/import-denue.ts` that:

a) Reads the CSV (use papaparse or csv-parse)
b) Filters to ONLY records where:
   - municipio is in MTY metro: Monterrey, Apodaca, Guadalupe, San Nicolás de los Garza,
     San Pedro Garza García, General Escobedo, Santa Catarina, García, Juárez, 
     Cadereyta Jiménez, Pesquería, Santiago, Salinas Victoria, El Carmen, 
     Ciénega de Flores
   - per_ocu indicates 11+ employees (parse the string: "0 a 5 personas" = skip,
     "6 a 10 personas" = skip, "11 a 30 personas" = include, etc.)

c) Parse per_ocu into employee_min/max/midpoint:
   - "11 a 30 personas" → 11, 30, 20
   - "31 a 50 personas" → 31, 50, 40
   - "51 a 100 personas" → 51, 100, 75
   - "101 a 250 personas" → 101, 250, 175
   - "251 y más personas" → 251, 500, 375

d) Compute lead_score (0-100) with these components:

   SCORE_SIZE (max 25):
   - 11-30 emp → 10
   - 31-50 → 15
   - 51-100 → 20
   - 101-250 → 25
   - 251+ → 22

   SCORE_RISK (max 25) based on SCIAN codigo_act first 2 digits:
   - Alto riesgo (25pts): 23 (construcción), 31-33 (manufactura), 
     62 (salud), 48-49 (transporte), 21 (minería)
   - Medio riesgo (15pts): 72 (alojamiento/alimentos), 56 (servicios), 
     43 (comercio), 46 (comercio al por menor), 81 (otros servicios)
   - Bajo riesgo (8pts): everything else

   SCORE_DIGITAL (max 15):
   - Has website (www not empty) → +5
   - Has phone (telefono not empty) → +5
   - Has email (correoelec not empty) → +5

   SCORE_ECONOMIC (max 15) based on estimated monthly contribution:
   Contribution = employee_midpoint × 10 × 113.14 × 0.08 (pesos/month)
   - > $50,000/mo → 15
   - > $30,000/mo → 12
   - > $15,000/mo → 9
   - > $5,000/mo → 6
   - else → 3

   PRIORITY:
   - score >= 70 → "Alta"
   - score >= 40 → "Media"
   - score < 40 → "Baja"

e) Map SCIAN codes to sector_group:
   - 23 → "Construcción"
   - 31,32,33 → "Manufactura"
   - 43 → "Comercio Mayoreo"
   - 46 → "Comercio Menudeo"
   - 48,49 → "Transporte"
   - 51 → "Información"
   - 52 → "Finanzas/Seguros"
   - 53 → "Inmobiliario"
   - 54 → "Servicios Profesionales"
   - 56 → "Servicios Apoyo"
   - 61 → "Educación"
   - 62 → "Salud"
   - 71 → "Entretenimiento"
   - 72 → "Alojamiento/Alimentos"
   - 81 → "Otros Servicios"
   - else → "Otros"

f) Map municipio to commercial_zone:
   - Monterrey → "Centro"
   - San Pedro Garza García → "Valle Oriente"
   - San Nicolás de los Garza, General Escobedo, Salinas Victoria → "Norte"
   - Guadalupe → "Oriente"
   - Apodaca, Pesquería → "Noreste"
   - Santa Catarina, García → "Poniente"
   - Juárez, Cadereyta Jiménez → "Sureste"
   - Santiago, El Carmen, Ciénega de Flores → "Sur"

g) Compute estimated_contribution_monthly = employee_midpoint × 10 × 113.14 × 0.08
   Recommend plan:
   - employee_midpoint <= 30 → "Impulsa" (6 UMAs/col)
   - employee_midpoint <= 100 → "Transforma" (10 UMAs/col)
   - else → "Lidera" (20 UMAs/col)

h) Map applicable NOMs by sector (store as JSON array):
   - Construcción: ["NOM-031-STPS","NOM-009-STPS","NOM-006-STPS"]
   - Manufactura: ["NOM-004-STPS","NOM-010-STPS","NOM-011-STPS"]
   - Salud: ["NOM-087-ECOL","NOM-005-STPS"]
   - Transporte: ["NOM-087-SCT","NOM-012-SCT"]
   - All: ["NOM-035-STPS","NOM-030-STPS","NOM-026-STPS","NOM-019-STPS"]

i) Build full_address by concatenating: tipo_vial + nom_vial + numero_ext + 
   (numero_int if present) + nomb_asent + municipio + cod_postal + entidad

j) Insert all records in batches of 500 using Drizzle ORM.
   Log progress: "Inserted batch X of Y (Z total records)"

STEP 3 — Create API endpoint POST /api/admin/denue/import that:
- Triggers the same import logic
- Returns { total_processed, total_imported, total_skipped, score_distribution }
- Requires admin auth

STEP 4 — Create API endpoint GET /api/admin/denue/stats that returns:
- total_prospects
- avg_score
- count_by_priority (Alta, Media, Baja)
- count_by_zone
- count_by_sector (top 10)
- count_by_pipeline_stage
- count_in_negotiation (pipeline_stage in ['Demo','Propuesta','Negociación'])
- count_clients (pipeline_stage = 'Cliente')

Run the import script after creating it. Target: ~21,000 records.
```

## PHASE 2: API ENDPOINTS FOR PROSPECTS + FILTERS
**Paste after Phase 1 data import completes.**

```
CONTEXT: Ceduverse CRM. The denue_prospects table is now populated with ~21K scored records.
Stack: React 19 + Express + Drizzle ORM + PostgreSQL.

TASK: Build comprehensive API endpoints for the Prospectos DENUE section.

1. GET /api/admin/denue/prospects — Main listing with filters
   Query params (all optional, combine with AND):
   - page (default 1), limit (default 50, max 200)
   - search (searches nom_estab, raz_social, nomb_asent, nom_vial)
   - zone (comma-separated: "Centro,Norte")
   - sector (comma-separated: "Manufactura,Construcción")
   - size (comma-separated ranges: "11-30,51-100,251+")
   - risk (comma-separated: "Alto,Medio")
   - score_min, score_max (0-100)
   - has_phone (boolean)
   - has_email (boolean)
   - has_web (boolean)
   - pipeline_stage (comma-separated or "null" for unassigned prospects)
   - assigned_socio_id
   - contact_group_id
   - sort_by (lead_score, nom_estab, employee_midpoint, municipio, estimated_contribution_monthly)
   - sort_dir (asc, desc — default desc for score)
   
   Response: { 
     data: [...prospects], 
     pagination: { page, limit, total, total_pages },
     aggregations: { total_filtered, avg_score, sum_estimated_contribution }
   }

2. GET /api/admin/denue/prospects/:id — Single prospect detail
   Returns full prospect record + interactions history + assigned socio info

3. PATCH /api/admin/denue/prospects/:id — Update prospect
   Updatable: pipeline_stage, assigned_socio_id, contact_group_id, notes

4. POST /api/admin/denue/prospects/bulk-update — Bulk operations
   Body: { prospect_ids: number[], action: string, value: any }
   Actions: 
   - "set_stage" → value = stage name
   - "assign_socio" → value = socio user id
   - "add_to_group" → value = group id
   - "remove_from_group" → removes group assignment

5. POST /api/admin/denue/prospects/:id/interactions — Add interaction
   Body: { interaction_type, description }

6. GET /api/admin/denue/prospects/:id/interactions — List interactions

7. CRUD for contact_groups:
   - GET /api/admin/denue/groups — List all groups with prospect counts
   - POST /api/admin/denue/groups — Create group
   - GET /api/admin/denue/groups/:id — Group detail + its prospects
   - PATCH /api/admin/denue/groups/:id — Update group
   - DELETE /api/admin/denue/groups/:id — Delete group (unassigns prospects)
   - POST /api/admin/denue/groups/:id/assign-socio — Assign socio to group

8. CRUD for saved_filters:
   - GET /api/admin/denue/filters — List saved filters
   - POST /api/admin/denue/filters — Save filter { name, filter_config }
   - DELETE /api/admin/denue/filters/:id — Delete saved filter

9. GET /api/admin/denue/export — Export filtered prospects as CSV
   Accepts same filter params as the main listing
   Returns CSV file download

All endpoints require admin authentication (check existing auth middleware pattern).
Use the existing Drizzle query patterns in the codebase.
```

# Prompt para Replit Agent — Blog Ceduverse

```
CONTEXTO: Ceduverse (ceduverse.org) — plataforma EdTech de capacitación laboral STPS 
en México. Stack: React 19 + Express + Drizzle ORM + PostgreSQL + Vite + Tailwind.
El proyecto ya tiene un admin dashboard, autenticación, y páginas de cursos.

TAREA: Crear un blog público completo en /blog. El blog es la pieza central de 
nuestra estrategia de content marketing — va a recibir ~35K visitantes/mes desde 
newsletters. Debe ser rápido, SEO-optimizado, y visualmente profesional.

====================================
PARTE 1: DATABASE SCHEMA (Drizzle ORM)
====================================

Crear estas tablas (NO borrar tablas existentes):

1. blog_posts:
   - id: serial primary key
   - title: varchar(200) NOT NULL
   - slug: varchar(200) NOT NULL UNIQUE
   - content_html: text NOT NULL
   - content_text: text NOT NULL
   - excerpt: varchar(300)
   - category: varchar(50) NOT NULL — valores: 'stps', 'fiscal', 'ia', 'cursos', 'casos'
   - target_sectors: jsonb DEFAULT '[]'
   - seo_keywords: jsonb DEFAULT '[]'
   - featured_image_url: varchar(500)
   - author_name: varchar(100) DEFAULT 'Ceduverse'
   - newsletter_subject: varchar(100)
   - newsletter_sent: boolean DEFAULT false
   - blog_views: integer DEFAULT 0
   - status: varchar(20) DEFAULT 'draft' — valores: 'draft', 'published', 'archived'
   - published_at: timestamp
   - created_at: timestamp DEFAULT now()
   - updated_at: timestamp DEFAULT now()

2. newsletter_subscribers:
   - id: serial primary key
   - email: varchar(255) NOT NULL UNIQUE
   - company_name: varchar(200)
   - sector: varchar(100)
   - municipio: varchar(100)
   - source: varchar(50) DEFAULT 'blog' — valores: 'blog', 'denue', 'webinar', 'referral'
   - status: varchar(20) DEFAULT 'active' — valores: 'active', 'unsubscribed', 'bounced'
   - subscribed_at: timestamp DEFAULT now()
   - unsubscribed_at: timestamp

Índices: slug (unique), category, status, published_at DESC, subscriber email (unique).
Ejecutar la migración después de crear el schema.

====================================
PARTE 2: API ENDPOINTS (Express)
====================================

Agregar estas rutas al servidor Express existente (revisar el patrón actual en routes.ts):

PÚBLICAS (sin auth):

GET /api/blog/posts
  - Query params: page (default 1), limit (default 9), category, search
  - Solo retorna posts con status='published'
  - Ordenar por published_at DESC
  - Response: { data: [...], pagination: { page, limit, total, totalPages } }

GET /api/blog/posts/:slug
  - Retorna post completo por slug
  - Incrementar blog_views en +1 (fire-and-forget, no bloquear response)
  - Solo si status='published'
  - 404 si no existe

GET /api/blog/categories
  - Retorna lista de categorías con count de posts publicados
  - Response: [{ category: 'stps', label: 'STPS y NOMs', count: 12 }, ...]

POST /api/newsletter/subscribe
  - Body: { email, company_name?, sector?, municipio? }
  - Validar email (regex básico)
  - Si ya existe y status='active' → 409 "Ya estás suscrito"
  - Si ya existe y status='unsubscribed' → reactivar
  - Response: { success: true, message: "¡Suscripción exitosa!" }

GET /api/newsletter/unsubscribe
  - Query: email (base64 encoded para no exponer en URL plain)
  - Marcar status='unsubscribed', unsubscribed_at=now()
  - Redirect a /blog?unsubscribed=true

ADMIN (requiere auth — usar el middleware existente):

POST /api/admin/blog/posts
  - Body: { title, slug, content_html, content_text, excerpt, category, 
    target_sectors, seo_keywords, featured_image_url, status }
  - Si status='published' y published_at es null → set published_at=now()
  - Response: post creado

PATCH /api/admin/blog/posts/:id
  - Actualizar cualquier campo
  - Si cambia a status='published' y published_at es null → set published_at=now()

DELETE /api/admin/blog/posts/:id
  - Soft delete: set status='archived'

GET /api/admin/blog/posts
  - Igual que el público pero retorna TODOS los status (draft, published, archived)
  - Incluye newsletter_sent, blog_views

GET /api/admin/newsletter/subscribers
  - Query: page, limit, status, source, search
  - Response con paginación

GET /api/admin/newsletter/stats
  - Response: { total_active, total_unsubscribed, by_source: {...}, by_sector: {...} }
```

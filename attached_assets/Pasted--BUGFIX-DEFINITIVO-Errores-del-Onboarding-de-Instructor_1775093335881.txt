## Bug 1 — VIDEO DE CÁMARA SE VE NEGRO (SIGUE SIN FUNCIONAR)

**Evidencia:** Screenshot 6 — El recuadro del video está completamente negro durante la grabación.

**El fix del `muted` NO se aplicó o no funcionó.** Verificar en `DigitalTwinRecorder.tsx`:

1. Buscar TODOS los `<video>` elements en el componente
2. Cada `<video>` que muestre el preview de la cámara DEBE tener exactamente estos atributos:

```tsx
<video
  ref={videoRef}
  autoPlay={true}
  playsInline={true}
  muted={true}
  style={{ transform: 'scaleX(-1)', background: '#000' }}
/>
```

**IMPORTANTE:** `muted` debe ser `muted={true}` como prop de React, NO como atributo HTML string. Verificar que no esté escrito como `muted="true"` (string) ni que esté dentro de un condicional que lo omita.

3. En la función `startCamera()`, FORZAR la reproducción:

```typescript
const startCamera = async () => {
  try {
    const stream = await navigator.mediaDevices.getUserMedia({
      video: { width: { ideal: 1280 }, height: { ideal: 720 }, facingMode: 'user' },
      audio: true,
    });
    streamRef.current = stream;
    
    if (videoRef.current) {
      videoRef.current.srcObject = stream;
      videoRef.current.muted = true;  // FORZAR muted via JS también
      
      // Esperar a que el metadata cargue y luego forzar play
      videoRef.current.onloadedmetadata = async () => {
        try {
          await videoRef.current?.play();
          console.log('[Camera] Preview playing successfully');
        } catch (playError) {
          console.error('[Camera] Play failed:', playError);
        }
      };
    }
  } catch (err) {
    console.error('[Camera] getUserMedia failed:', err);
  }
};
```

4. **Test:** Después de aplicar el fix, verificar que al hacer clic en "Empezar grabación" → "Iniciar grabación", se vea la cara del instructor en el recuadro del video.

## Bug 4 — "ERROR AL COMPLETAR" al hacer clic en "Ir a mi Panel"

**Evidencia:** Screenshot 8 — Toast rojo en esquina inferior derecha: "Error al completar. Intenta de nuevo"

**Problema:** El endpoint que marca el onboarding como completado está fallando. Esto bloquea al instructor de acceder a su panel.

**Fix:**
1. Verificar el endpoint `PATCH /api/instructor/onboarding` (o como se llame) que actualiza el `instructorOnboardingStep`
2. Agregar logging para ver el error exacto:

```typescript
// En el handler del botón "Ir a mi Panel":
const handleFinish = async () => {
  try {
    const res = await fetch('/api/instructor/onboarding', {
      method: 'PATCH',
      headers: {
        'Content-Type': 'application/json',
        Authorization: `Bearer ${token}`,
      },
      body: JSON.stringify({ step: 'completed' }),
    });
    
    const data = await res.json();
    console.log('[Onboarding] Complete response:', res.status, data);
    
    if (!res.ok) {
      throw new Error(data.error || data.message || `HTTP ${res.status}`);
    }
    
    // Navegar al panel
    navigate('/instructor');
  } catch (err) {
    console.error('[Onboarding] Error completing:', err);
    // Mostrar error detallado, no solo "Error al completar"
    toast.error(`Error: ${err.message}`);
  }
};
```

3. **Fallback crítico:** Si el endpoint falla, **DEJAR PASAR AL INSTRUCTOR DE TODOS MODOS.** No bloquear el acceso al panel por un error de onboarding. El onboarding es informativo, no un gate.

```typescript
// SIEMPRE navegar al panel, independientemente del error:
const handleFinish = async () => {
  try {
    await fetch('/api/instructor/onboarding', {
      method: 'PATCH',
      headers: { ... },
      body: JSON.stringify({ step: 'completed' }),
    });
  } catch (err) {
    console.warn('[Onboarding] Could not mark as complete, but proceeding anyway:', err);
  }
  // SIEMPRE navegar
  navigate('/instructor');
};
```

4. Si el Panel de Instructor verifica que el onboarding esté completado para dejar entrar, **eliminar esa verificación.** El panel debe ser accesible siempre que `is_instructor = true`.

## Bug 6 — CÁMARA NO SE APAGA

**Evidencia:** El LED de la cámara sigue encendido después de grabar.

**Fix:** En `DigitalTwinRecorder.tsx`, asegurar que `stopCamera()` detenga TODOS los tracks:

```typescript
const stopCamera = () => {
  // Detener todos los tracks del stream
  if (streamRef.current) {
    const tracks = streamRef.current.getTracks();
    tracks.forEach(track => {
      track.stop();
      console.log(`[Camera] Stopped track: ${track.kind}`);
    });
    streamRef.current = null;
  }
  
  // Limpiar el elemento video
  if (videoRef.current) {
    videoRef.current.srcObject = null;
    videoRef.current.load(); // Forzar liberación del stream
  }
};
```

Llamar `stopCamera()` en:
- Al terminar de grabar (en el `onstop` del MediaRecorder)
- Al pasar de review a processing
- Al hacer clic en "Omitir" en cualquier paso
- Al hacer clic en "Siguiente" si se omite el Digital Twin
- En el `useEffect` cleanup: `return () => { stopCamera(); }`

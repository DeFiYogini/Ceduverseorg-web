# Prompt para Replit Agent — API de Empresas para MeCorrieron.mx

Pegar esto completo en el Replit Agent de Ceduverse (ceduverse.replit.app):

```
CONTEXTO: Ceduverse tiene una base de datos propietaria de ~21,000+ empresas en la
zona metropolitana de Monterrey (tabla denue_prospects en PostgreSQL). Cada registro 
tiene: nombre, razón social, sector SCIAN, municipio, colonia, dirección, teléfono, 
email, sitio web, latitud/longitud, rango de empleados, y un scoring propietario.

Tenemos otra plataforma: MeCorrieron.mx (mecorrieron.mx) — una app de derechos 
laborales que incluye un "Radar de Ambiente Laboral" (mapa tipo Google Maps donde 
trabajadores ven y califican empresas como fuentes de trabajo). MeCorrieron necesita 
una BASE de empresas reales con ubicación geográfica para poblar su Radar.

TAREA: Crear una API REST segura en el servidor Express de Ceduverse que exponga 
datos de empresas para consumo externo de MeCorrieron.mx. La API debe:
1. Estar protegida con API key (no pública)
2. Devolver datos LIMITADOS (no exponer scoring, fuente de datos, ni campos internos)
3. Ser eficiente para carga por viewport del mapa (consultas por bounding box)
4. Tener rate limiting
5. NUNCA exponer: lead_score, priority, estimated_contribution_monthly, 
   pipeline_stage, assigned_socio_id, clee, codigo_act, ni campos propietarios

NOMENCLATURA: En los responses de la API, los datos se presentan como procedentes
de "Directorio Empresarial Ceduverse" — NUNCA mencionar DENUE, INEGI, ni la fuente
real de los datos.

=============================================
PARTE 1: TABLA DE API KEYS
=============================================

Crear tabla para gestionar acceso externo:

api_keys:
  - id: serial primary key
  - key_hash: varchar(255) NOT NULL  -- bcrypt hash de la API key
  - name: varchar(100) NOT NULL      -- "mecorrieron-production", "mecorrieron-staging"
  - owner: varchar(100)              -- "MeCorrieron.mx"
  - permissions: jsonb DEFAULT '["read:companies"]'
  - rate_limit_per_minute: integer DEFAULT 60
  - rate_limit_per_day: integer DEFAULT 10000
  - allowed_origins: jsonb DEFAULT '["https://mecorrieron.mx", "http://localhost:3000"]'
  - is_active: boolean DEFAULT true
  - requests_today: integer DEFAULT 0
  - requests_today_reset_at: timestamp
  - last_used_at: timestamp
  - created_at: timestamp DEFAULT now()
  - expires_at: timestamp  -- null = no expira

Generar una API key inicial para MeCorrieron:
- Generar un UUID v4 como raw key (ej: "cv_live_a3f7b2c1d4e5f6...")
- Guardar el hash bcrypt en la tabla
- Loggear la key en consola UNA vez para copiarla (no guardarla en plain text)
- Nombre: "mecorrieron-production"
- Rate limit: 120/minuto, 50,000/día

=============================================
PARTE 2: MIDDLEWARE DE AUTENTICACIÓN API
=============================================

Crear middleware: authenticateApiKey

Lógica:
1. Leer header: Authorization: Bearer cv_live_xxxxx
2. Si no hay header → 401 { error: "API key requerida" }
3. Buscar en api_keys donde is_active=true
4. Comparar con bcrypt contra todos los key_hash activos
5. Si no match → 403 { error: "API key inválida" }
6. Si expirada → 403 { error: "API key expirada" }
7. Verificar Origin header contra allowed_origins (si hay Origin)
8. Verificar rate limit:
   - Si requests_today >= rate_limit_per_day → 429 "Límite diario alcanzado"
   - Implementar rate limit por minuto con un Map en memoria
9. Incrementar requests_today y last_used_at
10. Resetear requests_today si requests_today_reset_at < hoy
11. Adjuntar apiKey al req para logging

=============================================
PARTE 3: ENDPOINTS DE LA API
=============================================

Base path: /api/v1/external/companies
Todos requieren authenticateApiKey middleware.

=============================================
PARTE 4: SEGURIDAD Y RATE LIMITING
=============================================

1. Rate limiting en memoria (Map):
   - Por API key: max requests_per_minute
   - Global: max 1000 req/minuto total
   - Responder con 429 + Retry-After header

2. Response headers:
   X-RateLimit-Limit: 120
   X-RateLimit-Remaining: 85
   X-RateLimit-Reset: 1711929600
   X-Source: Directorio Empresarial Ceduverse
   Cache-Control: public, max-age=3600  (cachear 1 hora)

3. CORS:
   Permitir solo los origins de la API key (allowed_origins)
   
4. Request logging:
   Crear tabla api_request_logs:
   - id, api_key_id, endpoint, method, query_params (jsonb), 
     response_status, response_time_ms, ip_address, user_agent, created_at
   Log CADA request para analytics y debugging.
   Cleanup: eliminar logs > 30 días con un cron.

5. ID Ofuscación:
   Los IDs expuestos en la API NO son los IDs internos directos.
   Usar formato: "ceduverse-{id}" donde {id} es el serial de denue_prospects.
   En el endpoint de detalle, parsear el prefijo antes de buscar.
   (Opción más segura: usar hashids para ofuscar el ID numérico)
```

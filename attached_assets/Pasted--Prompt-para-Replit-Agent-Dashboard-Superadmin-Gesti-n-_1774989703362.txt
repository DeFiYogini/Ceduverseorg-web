--- 2.1: CRUD de API Keys ---

GET /api/admin/apis/keys
  Response: lista de API keys (SIN el hash, solo metadata)
  [{
    id, name, owner, permissions, rate_limit_per_minute, rate_limit_per_day,
    allowed_origins, is_active, requests_today, last_used_at, created_at,
    expires_at, total_requests_30d (subquery de api_request_logs)
  }]

POST /api/admin/apis/keys
  Body: { name, owner, permissions?, rate_limit_per_minute?, rate_limit_per_day?,
    allowed_origins?, expires_at? }
  Lógica:
    1. Generar raw key: "cv_live_" + crypto.randomUUID().replace(/-/g, '')
    2. Hash con bcrypt (10 rounds)
    3. Guardar hash en DB
    4. Retornar la raw key UNA SOLA VEZ en la response
  Response: { 
    id, name, owner, 
    raw_key: "cv_live_a3f7b2c1d4e5f6...",  // SOLO SE MUESTRA ESTA VEZ
    message: "Copia esta clave ahora. No se puede recuperar después."
  }

PATCH /api/admin/apis/keys/:id
  Body: campos a actualizar (name, owner, rate limits, origins, is_active, expires_at)
  NO se puede cambiar el key_hash

DELETE /api/admin/apis/keys/:id
  Soft delete: set is_active = false (no borrar, por auditoría)
  Response: { success: true, message: "API key desactivada" }

POST /api/admin/apis/keys/:id/regenerate
  Genera nueva raw key + nuevo hash para la misma API key (rotación)
  Response: { raw_key: "cv_live_nuevo...", message: "Key regenerada" }
